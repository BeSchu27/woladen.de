Live API Lasttest

Misst Frontend-Laufzeiten, liest Server-Timing aus und trennt so Browser-Decode, Netz, Payload-Bau und SQLite-Zeit.

Request-Vorschau

Das ist der konkrete Request, der mehrfach gegen die Live-API geschickt wird.

Verdichtete Laufzeiten

Mittelwert, Median und p95 für die wichtigsten Phasen.

Einzelläufe

Jeder Lauf mit HTTP-Status, Gesamtdauer und den relevanten Teilzeiten.

Interpretation

Worauf die Werte hindeuten.

  • Hohe SQLite Query-Zeit bei niedriger JSON encode-Zeit spricht für Datenbank- oder Query-Plan-Themen.
  • Hohe JSON encode- oder JSON parse-Zeit spricht eher für Payload-Größe oder unnötige Felder.
  • Große Lücke zwischen Bis Header und Server gesamt deutet eher auf Netz, TLS oder Queuing.
  • Wenn Resource TTFB leer bleibt, fehlt meist Timing-Allow-Origin auf dem API-Response.
